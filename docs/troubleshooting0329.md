## 1. 배경 — trust.py가 하는 일

기사 본문을 Gemini에 넣으면, 5가지 기준으로 점수를 매깁니다.

| 기준 | 설명 |
|------|------|
| 출처 신뢰성 | 언론사가 믿을 만한 곳인가 |
| 근거 지지도 | 전문가 인용, 통계 등 근거가 있는가 |
| 문체 중립성 | 감정적·선동적 표현이 없는가 |
| 논리 일관성 | 주장과 결론이 서로 맞는가 |
| 어뷰징 위험 | 제목이 본문과 동떨어진 낚시성인가 |

이 5개를 가중 합산해서 **0~100점**을 내고, 70 이상이면 `likely_true`, 40 미만이면 `likely_false`를 반환합니다.

---

## 2. 문제 — "이 점수가 실제로 맞는가?"

점수 계산 공식은 만들었지만, **"진짜 가짜뉴스에 낮은 점수가 나오는가"를 한 번도 테스트하지 않은 상태**였습니다.

이걸 검증하려면 `정답 레이블`이 필요합니다.
- 기사 A → 실제로 신뢰할 수 있음 (label = 1)
- 기사 B → 실제로 가짜뉴스 (label = 0)

이런 레이블이 붙은 한국어 기사 데이터셋을 찾는 것이 첫 번째 과제였습니다.

---

## 3. 1차 시도 — AI-HUB 낚시성 기사 데이터셋

**AI-HUB**에서 제공하는 "낚시성 기사" 데이터셋(약 10만 건)으로 테스트했습니다.

결과:

| 그룹 | trust.py 점수 평균 |
|------|------------------|
| Real (정상 기사) | **46점** |
| Fake (낚시성 기사) | **60점** |

정상 기사가 더 낮고, 가짜 기사가 더 높게 나왔습니다. **완전히 반대 방향**입니다.

**원인 분석:**

AI-HUB의 `Fake` 레이블은 **"제목이 낚시성인 기사"** 를 의미합니다. 본문이 거짓이라는 뜻이 아닙니다.

```
Fake 기사 실제 본문 예시:
"에너지 수입 급증으로 이달 무역적자 21억달러 기록,
 원유 57.8%, 가스 114.3% 증가…"

→ 본문에 구체적 수치, 통계 풍부
→ trust.py는 이 본문을 "근거가 충실한 기사"로 평가 → 높은 점수
```

trust.py는 **본문 품질**을 보는데, 이 데이터셋은 **제목 선정성**에 레이블이 붙어 있었습니다. 맞지 않는 조합이었습니다.

**결론**: 데이터셋 교체 필요.

---

## 4. 2차 시도 — SNU 팩트체크 데이터

서울대 언론정보연구소가 운영하던 **SNU 팩트체크** 서비스의 판정 데이터(4,762건)를 발견했습니다.

```
제목: "액상형 대마는 합법이다"
판정: 사실 아님
```

이런 구조로 되어 있어, `사실` = label 1, `사실 아님` = label 0 으로 쓸 수 있을 것 같았습니다.

**문제**: 제목과 판정만 있고 **기사 본문이 없습니다.** SNU 팩트체크 서비스 자체가 종료되었기 때문입니다.

**대응**: 제목을 네이버에서 검색해서 기사 본문을 직접 크롤링하기로 했습니다.

---

## 5. 크롤링 파이프라인 구축

`crawl_exp/` 폴더에 4단계 파이프라인을 만들었습니다.

```
01_fetch_data.py     → SNU 데이터 다운로드
02_parse_sources.py  → 출처 필드를 유형별로 분류
03_search_and_crawl.py → 네이버 검색 API로 기사 본문 수집
04_build_eval.py     → 수집 결과를 평가용 파일로 조립
```

**크롤링 결과 (TYPE_A 178건 기준):**
- 성공률 약 60% (~107건 본문 확보)
- 유사도(sim) 0.3 이상 기준으로 신뢰할 수 있는 매칭

---

## 6. 2차 평가 결과 및 핵심 발견

수집한 기사로 다시 평가를 돌렸습니다.

| 그룹 | trust.py 점수 평균 | 목표 |
|------|------------------|----|
| 신뢰 그룹 (사실 판정) | 64.3점 | ≥ 65 |
| 불신 그룹 (사실아님 판정) | **60.6점** | ≤ 45 |

여전히 불신 그룹 점수가 높습니다. 이번엔 데이터셋 문제가 아니라 **더 근본적인 이유**가 있었습니다.

**SNU "사실 아님" 기사의 실제 모습:**

```
기사: "정부, 내년부터 인공눈물 가격 최대 10배 인상 예정"
출처: 연합뉴스
작성: 전문가 인용 포함, 중립적 문체, 통계 수치 기재
팩트체크 판정: 사실 아님 (수치가 틀렸음)
```

연합뉴스 기자가 쓴 **잘 쓰인 기사**인데 수치가 틀린 겁니다.
trust.py 입장에서는 "근거도 있고, 출처도 좋고, 문체도 중립적"이므로 **당연히 높은 점수**를 줍니다.

---

## 7. 핵심 결론 — 목적 재정의

이 분석을 통해 trust.py가 측정하는 것과 "사실 여부"는 다른 개념임이 명확해졌습니다.

| | trust.py가 탐지하는 것 | 팩트체크가 판정하는 것 |
|--|----------------------|---------------------|
| 연합뉴스가 수치 틀린 기사 | ✅ 높은 점수 (잘 쓰임) | ❌ 사실 아님 |
| 출처불명 + 선동적 루머 기사 | ❌ 낮은 점수 | ❌ 사실 아님 |
| 통계·전문가 인용 풍부한 기사 | ✅ 높은 점수 | ✅ 사실 |

**trust.py는 "사실 판별기"가 될 수 없습니다.**
이것은 버그가 아니라 설계 범위의 문제입니다.

그러나 trust.py가 **잘 탐지하는 것**이 분명히 있습니다:
- 근거 없이 주장만 나열하는 기사
- 자극적 제목으로 클릭을 유도하는 기사
- 감정적·선동적 언어를 쓰는 기사
- 출처 불명의 기사

이것들은 모두 **가짜뉴스의 전형적인 구조적 패턴**입니다.

**→ 목적 재정의: "사실 여부 판별"이 아닌 "가짜뉴스 패턴 경고"**

---

## 8. 앞으로의 방향

### 로직

- 가중치를 "패턴 탐지" 목적에 맞게 재조정
  - 문체 중립성, 어뷰징 위험도 비중 상향
- 기존 사실/사실아님 데이터 대신, **수작업 대조 쌍 테스트**로 검증
  - "근거 있는 기사 vs 동일 기사에서 인용 제거" → 점수 차이 확인

### UX

- 점수를 "이 기사는 67점입니다" 대신 기준별 경고로 표시
- "사실 여부를 보장하지 않습니다" 면책 문구 고정 노출
- verdict 텍스트를 "신뢰 가능/불가"가 아닌 "패턴 정상/패턴 주의" 언어로 변경

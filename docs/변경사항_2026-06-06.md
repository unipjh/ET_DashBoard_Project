# 변경사항 2026-06-06

## 1. 첫 화면과 기사 목록 UX

### 1. 시작 화면 대기 시간 단축
- 기존 상태: 앱 시작 화면이 고정 시간 동안 오래 유지되는 구조.
- 변경 상태: 첫 방문만 짧게 표시, 같은 탭 재방문 시 거의 즉시 종료.
- 사용자 효과: “로딩 화면에서 멈춤” 느낌 감소.

```text
기존 흐름
앱 진입 -> 고정 스플래시 대기 -> 메인 화면

변경 흐름
앱 진입 -> 짧은 스플래시 -> 메인 화면
재방문 -> 최소 스플래시 -> 메인 화면
```

### 2. 추천 기사 로딩과 최신 기사 로딩 분리
- 기존 상태: 추천 기사 요청이 늦으면 메인 목록 전체 표시도 함께 지연.
- 변경 상태: 최신 기사 목록과 추천 기사 요청을 독립적으로 처리.
- 사용자 효과: 추천 영역이 늦어도 핵심 기사 탐색 가능.

```text
기존 흐름
추천 기사 로딩 + 최신 기사 로딩 완료 -> 화면 표시

변경 흐름
최신 기사 로딩 완료 -> 목록 먼저 표시
추천 기사 로딩 완료 -> 추천 영역 별도 갱신
```

### 3. 목록 로딩 skeleton 추가
- 기존 상태: 데이터 도착 전 단순 로딩 문구 중심의 대기 화면.
- 변경 상태: 기사 카드가 들어올 위치에 skeleton UI 표시.
- 사용자 효과: 화면 구조 예측 가능, 레이아웃 흔들림 감소.

```text
로딩 중 화면
[검색창]
[카테고리]
[기사 카드 자리 skeleton]
[기사 카드 자리 skeleton]
[기사 카드 자리 skeleton]
```

### 4. 기사 목록 오류 상태 개선
- 기존 상태: API 실패 시 사용자가 원인을 파악하기 어려운 상태.
- 변경 상태: 오류 카드, 안내 문구, 다시 시도 버튼 표시.
- 사용자 효과: 장애 상황에서도 다음 행동 명확화.

```text
API 실패
  -> "기사를 불러오지 못했습니다."
  -> "네트워크나 서버 상태를 확인..."
  -> [다시 시도]
```

### 5. 빈 상태별 다음 행동 제공
- 기존 상태: 검색 결과 없음, 기사 없음, 오류 상태가 비슷하게 보이는 구조.
- 변경 상태: 검색 결과 없음은 필터 초기화, 기사 없음은 관리자 이동으로 연결.
- 사용자 효과: “아무것도 없음” 상태에서 다음 단계 제시.

```text
검색 결과 없음 -> [필터 초기화]
전체 기사 없음 -> [관리자 화면으로 이동]
서버 오류 -> [다시 시도]
```

## 2. 관리자 인증과 보호

### 1. 관리자 로그인 화면 추가
- 기존 상태: `/admin` 접속 시 통계와 작업 버튼이 바로 노출.
- 변경 상태: 비밀번호 입력 화면을 먼저 표시.
- 사용자 효과: 관리자 기능 진입 경계 명확화.

```text
/admin 진입
  -> 관리자 인증 화면
  -> 비밀번호 확인
  -> 관리자 대시보드
```

### 2. 서버 비밀번호 미설정 상태 분리
- 기존 상태: 비밀번호 설정 누락과 오입력이 같은 실패처럼 보일 위험.
- 변경 상태: `ADMIN_PASSWORD` 미설정 시 별도 메시지 표시.
- 운영 효과: `.env` 설정 누락을 즉시 식별 가능.

```text
ADMIN_PASSWORD 없음
  -> 관리자 API 503
  -> "Admin password is not configured."
  -> 서버 환경변수 확인 필요
```

### 3. 관리자 비밀번호 저장 방식 변경
- 기존 상태: 브라우저 저장소에 비밀번호를 둘 경우 노출 위험 존재.
- 변경 상태: 인증 상태와 비밀번호를 현재 탭 메모리에서만 유지.
- 보안 효과: 새로고침 또는 탭 종료 시 인증 정보 잔존 최소화.

```text
저장 위치
localStorage     사용 안 함
sessionStorage   사용 안 함
탭 메모리        현재 인증 중에만 사용
```

### 4. 관리자 API 서버 측 보호
- 기존 상태: 화면에서만 막아도 API를 직접 호출할 수 있는 위험.
- 변경 상태: 관리자 API가 직접 `X-Admin-Password` 헤더 확인.
- 보안 효과: 화면 우회 호출 방지.

```text
관리자 요청
  -> X-Admin-Password 확인
  -> 일치: 처리
  -> 불일치/누락: 401
  -> 서버 설정 없음: 503
```

### 5. 관리자 작업 전체 보호
- 기존 상태: 통계, 크롤링, 분석, 중복 제거, 키워드 추출의 보호 기준 불명확.
- 변경 상태: 조회성 기능과 데이터 변경성 기능 모두 인증 대상.
- 운영 효과: 외부 요청에 의한 작업 실행 위험 감소.

```text
보호 대상
- 통계 조회
- 크롤링 시작
- 신뢰도 분석
- 중복 제거
- 키워드 추출
- 작업 상태 조회
```

## 3. 백엔드 안정성과 장애 응답

### 1. `/health` 상태 점검 추가
- 기존 상태: 서버 실행 여부와 DB 사용 가능 여부를 분리 확인하기 어려운 구조.
- 변경 상태: `/health`에서 서버와 DB 상태를 함께 반환.
- 운영 효과: 장애 원인 분리 속도 향상.

```text
정상
GET /health -> 200
{ status: "ok", database: "ok" }

DB 장애
GET /health -> 503
{ status: "degraded", database: "unavailable" }
```

### 2. DB 초기화 실패 기록
- 기존 상태: 앱 시작 중 DB 초기화 실패 시 원인 추적이 늦어지는 문제.
- 변경 상태: 시작 단계의 DB 실패 원인을 앱 상태와 로그에 기록.
- 운영 효과: 배포 직후 장애 확인 지점 명확화.

```text
서버 시작
  -> DB 초기화 시도
  -> 성공: 정상 시작
  -> 실패: 원인 기록 + /health에서 확인
```

### 3. 기사 목록 DB 장애 응답 정리
- 기존 상태: DB 장애가 기사 목록 API에서 500으로 번질 수 있는 구조.
- 변경 상태: 기사 목록 DB 장애를 503으로 반환.
- 사용자 효과: 프론트에서 오류 카드와 다시 시도 버튼으로 연결 가능.

```text
GET /api/articles
  -> DB 정상: 기사 목록 반환
  -> DB 장애: 503 반환
  -> 프론트 오류 카드 표시
```

### 4. 관리자 통계 DB 장애 응답 정리
- 기존 상태: 인증은 성공해도 DB 장애가 일반 서버 오류로 보일 위험.
- 변경 상태: 관리자 통계 DB 장애도 503으로 정리.
- 운영 효과: 인증 문제와 DB 문제 구분 가능.

```text
GET /api/admin/stats
  -> 인증 실패: 401
  -> 서버 비밀번호 없음: 503
  -> DB 장애: 503
  -> 정상: 통계 반환
```

### 5. 선택 의존성 fallback 처리
- 기존 상태: 일부 패키지가 없으면 앱 import 자체가 실패하는 문제.
- 변경 상태: DB/검색 관련 패키지 누락 시 앱은 뜨고 기능별 오류로 분리.
- 개발 효과: 로컬 점검과 smoke test 수행 가능.

```text
패키지 누락
  -> 앱 import 실패  X
  -> 기능 사용 시 명확한 실패 O
  -> /health 또는 테스트로 원인 확인
```

- 필요 작업: 새 Google Gen AI SDK 기준으로 호출부 교체.
- 기대 효과: 장기 유지보수 위험 감소.

```text
현재
google.generativeai -> deprecation warning

개선
새 SDK로 교체 -> 경고 제거 + 최신 API 대응
```

### 2. 관리자 작업 상태 영구 저장
- 현재 상태: 작업 상태가 주로 서버 메모리에 의존.
- 필요 작업: 작업 ID, 시작/종료 시간, 실패 원인을 DB에 기록.
- 기대 효과: 서버 재시작 후에도 작업 이력 추적 가능.

```text
현재
서버 메모리 상태 -> 재시작 시 소실

개선
DB 작업 기록 -> 재시작 후에도 조회 가능
```

### 3. 성능 로그 대시보드
- 현재 상태: 프론트 성능 로그를 남길 기반만 존재.
- 필요 작업: 성능 이벤트를 모아 보는 운영 화면 추가.
- 기대 효과: 실제 사용자 체감 성능 추적 가능.

```text
현재
성능 이벤트 저장 기반

개선
운영 대시보드
  -> LCP/CLS/입력 지연 추이 확인
```

### 4. UI 자동 테스트 확대
- 현재 상태: 주요 화면은 브라우저로 수동 검증.
- 필요 작업: Playwright 자동 테스트로 메인/상세/관리자/오류 상태 고정.
- 기대 효과: 화면 회귀를 자동으로 감지.

```text
우선 자동화 후보
- 메인 목록
- 상세 화면
- 관리자 로그인
- 오류 카드
- 모바일 레이아웃
```

### 5. 환경변수 점검 UX 개선
- 현재 상태: 설정 누락은 API 응답이나 로그인 메시지로 확인.
- 필요 작업: 필수 환경변수 상태를 한눈에 보는 점검 화면 또는 명령 추가.
- 기대 효과: 배포 전 설정 누락 발견 속도 향상.

```text
점검 대상
DATABASE_URL
ADMIN_PASSWORD
GEMINI_API_KEY
ALLOWED_ORIGINS
VITE_API_URL
```

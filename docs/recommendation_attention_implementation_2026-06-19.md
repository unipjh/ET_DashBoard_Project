# 추천 시스템 고도화 정리

작성일: 2026-06-19

## 핵심 흐름

1. 노출 기록
   - 메인 화면에 기사 목록이 보이면 `impression` 로그를 남긴다.
   - 저장 내용: 기사 ID, 보인 위치, 페이지, 카테고리, 검색어.
   - 활용 목적: "봤지만 클릭하지 않은 기사"를 찾아 추천 학습의 반대 예시로 쓴다.

2. 오늘의 추천 뉴스
   - 기존처럼 전체 기사 중 신뢰도와 최신성을 기준으로 보여준다.
   - 화면 왼쪽 영역에 표시된다.

3. 당신을 위한 뉴스
   - 화면 오른쪽 영역에 개인화 추천 5개를 보여준다.
   - 클릭하거나 상세 페이지를 본 기사를 바탕으로 다시 계산한다.
   - 로그인하지 않아도 브라우저 세션 기준으로 동작한다.

4. 추천 계산 순서
   - 행동 기록이 없으면 최신 기사로 보여준다.
   - 행동 기록이 1개 이상 있으면 개인화 추천을 시도한다.
   - 학습된 attention 모델이 있으면 attention 추천을 먼저 사용한다.
   - 모델이 아직 없으면 기존 임베딩 유사도 추천을 사용한다.
   - 임베딩이 부족하면 최근 본 기사와 같은 카테고리의 최신 기사를 보여준다.
   - 모든 방법이 실패하면 마지막으로 전체 최신 기사를 보여준다.

## 이번에 바로잡은 부분

- 추천 화면이 같은 5개를 계속 보여주던 원인을 줄였다.
- 프론트가 추천 API 결과를 오래 붙잡지 않게 바꿨다.
- 메인으로 돌아올 때 추천 API를 다시 부르게 했다.
- 로그인 ID와 세션 ID 중 하나만 맞아도 행동 기록을 읽게 했다.
- 행동 5개를 기다리지 않고, 1개부터 개인화 추천을 시도하게 했다.
- 초기 DB처럼 임베딩이 비어 있어도 카테고리 기반 추천으로 변화를 만들게 했다.

## 수동 확인 시나리오

1. 메인 화면을 연다.
2. 오른쪽 `AI가 분석한 당신을 위한 뉴스` 5개를 확인한다.
3. 아무 기사나 클릭해 상세 페이지로 들어간다.
4. 다시 메인 화면으로 돌아온다.
5. 추천 API가 다시 호출되는지 브라우저 Network 탭에서 확인한다.
6. 행동 기록이 쌓이면 오른쪽 추천 5개가 최신순 고정에서 벗어나는지 확인한다.

## 확인할 API

- 추천 API: `GET /api/recommendations?session_id=...&user_id=...&limit=5`
- 로그 API: `POST /api/logs`
- 추천 판단 기준:
  - 응답이 매번 최신 기사만 나오면 아직 행동 기록이나 임베딩이 부족한 상태다.
  - 최근 본 기사와 같은 카테고리 기사가 나오면 초기 개인화 보강이 동작 중이다.
  - 학습 가중치가 배포되면 attention 추천이 우선 적용된다.
